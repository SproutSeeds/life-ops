# Daily and Weekly Flow

This system is built around a simple idea: the agenda should show both hard commitments and the routines that keep life from drifting.

## Daily flow

- `08:15` GitHub morning sweep: collect notifications, live PRs, review states, and repo action items into LifeOps
- `08:20` ORP project sweep: collect the highest-level action for each starred, super-starred, or pinned project map
- `08:30` morning planning: print or review the FRG-first day sheet, scan the next 7 days, pick the top priorities, and confirm any time-sensitive follow-ups
- `10:00` day-sheet print: rerun GitHub and ORP refreshes, save a print snapshot, and print the current FRG-first day sheet
- `11:30` communications sweep: reply, delegate, or schedule a follow-up time instead of carrying open loops in your head
- `17:30` shutdown: close the day by checking tomorrow, capturing loose commitments, and marking completed follow-ups done

## Weekly flow

- Monday: weekly planning and constraint setting
- Midweek: quick load check so the second half of the week stays realistic
- Sunday evening: review the coming week, move stale follow-ups forward, and make sure key relationships or organizations are not neglected

## Operating principle

The weekly agenda should become the default view because it answers the questions that matter:

- what is fixed on the calendar?
- what communications need attention?
- which GitHub and ORP project actions deserve attention today?
- what routines should happen even when the week gets noisy?
- which organizations or people need a touchpoint soon?
